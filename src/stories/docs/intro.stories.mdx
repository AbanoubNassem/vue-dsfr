# Qu'est-ce que VueDsfr

VueDsfr est un portage du [Système de design français](https://www.systeme-de-design.gouv.fr/)
en [Vue 3](https://v3.vuejs.org/) sous forme de bibliothèque de composants.

# Pourquoi VueDsfr

Le Bureau des ressources et réalisations (BRR) a choisi sa stack technique pour le front, et elle inclut
Vue 3. Le Système de design français (DSFR) est développé en JavaScript natif et en SCSS (SASS).

Le BRR a donc besoin d'un portage en Vue 3 de ce DSFR pour ses futurs projets qui devront le respecter.

Il existe déjà un autre [dépôt](https://github.com/ac-lyon-dev/vue-gouv-fr) avec un début d'implémentation.
Néanmoins, il ne correspond pas aux standards que le BRR s'impose.

# Quels sont les principes et les buts de VueDsfr

Ce projet de bibliothèque de composants

- doit être fait avec Vue 3
- doit être utilisable facilement en tant que plugin
- doit être utilisable facilement dans un projet npm
- doit mettre à disposition deux types de documentation :
  - une pour aider au développement
  - une autre pour l'utilisation
- doit publier un site complet avec les composants et leurs variants testables dans celui-ci
- doit avoir un site déployé avec ce storybook disponible pour tous
- doit avoir une couverture de test minimale
- doit contenir le moins de CSS global possible et le CSS scopé inclus dans les composants
- doit mettre à disposition les composants individuellement sans avoir à importer toute la bibliothèque
  (pour les petits projets)

# Quels sont les choix techniques retenus

Le serveur de développement est [Vite](https://vitejs.dev).

Le CSS est écrit avec [PostCSS](https://postcss.org/) et notamment le plugin [postcss-preset-env](https://preset-env.cssdb.org/).

Toute la documentation est faite avec [Storybook](https://storybook.js.org/), et le site pour tester les composants
est le storybook lui-même.

Les tests sont faits avec [Jest](https://jestjs.io) et dans un futur proche avec [Cypress](https://www.cypress.io/).
